CFP fit analysis — each topic evaluated #cfp-fit

The CFP names four primary topics and ~5 welcome topics. Each is rated against this repo's existing research threads: PA-AKG (provenance-aware agentic KG, hybrid retrieval), AKG-E (engineering KGs with VVUQ — just submitted to Amazon-VT), SVF (structured validation framework), and the Brown lab SE thread (GenAI tool evaluation for software engineering: AutoPyDep, AutoPrint, SE belief auditing).

Topic 1 — No-code / low-code agentic AI solutions
Primary · CFP headline
Weak
What it asks for: Platforms letting non-developers deploy and manage AI agents at scale.
Fit: Nothing in the portfolio is no-code / low-code. PA-AKG, AKG-E, SVF are research-grade developer-facing systems. Force-fitting AKG-E's typed-interface framing as a "low-code spec for domain experts" is visible as topic mismatch.
Verdict: Skip — not primary, not secondary.
Topic 2 — AI-enhanced productivity applications

Moderate

Fit: Moderate via the Brown lab SE thread — AutoPyDep (Python dependency mgmt, FSE '25), AutoPrint (debugging, VL/HCC '25), SQL-LLM eval (VL/HCC '24). But the work evaluates productivity tools rather than building new ones. Productivity as primary would require a fresh build commitment.
Verdict: Viable secondary only.
Topic 3 — Multi-agent systems and collaborations
Primary · CFP headline · Recommended primary
Strong
What it asks for: Agents working together securely and effectively — orchestration, coordination, governance.
Fit: Three direct matches.
  • PA-AKG is multi-agent by construction — retrieval, KG-curation, validation agents, with provenance routed between them.
  • AKG-E orchestrates domain-ontology + simulation + VVUQ-validation agents.
  • SVF evaluates multi-agent systems with structured validation criteria (meta-fit but defensible).
Novelty wedge: provenance-aware coordination — every inter-agent message carries verifiable evidence chains, addressing Topic 3's "securely and effectively" qualifier directly.
Verdict: Strongest primary candidate.

Related work landscape (2024–2026) #related-work

Active field. Multi-agent + GraphRAG + agentic-RAG has been the dominant 2025 research thread. The novelty wedge for PA-AKG is that no existing system treats per-claim provenance as a typed, propagated, verifiable artifact across multi-agent message passing — the closest comparator is GraphTracer (failure attribution, after-the-fact), and the SoK March 2026 survey explicitly lists "oversight mechanisms for autonomous loops" as an open gap.

Paper Year What it does How PA-AKG differs
SoK: Agentic RAG — Taxonomy & Research Directions
arXiv 2603.07379		 Mar 2026 Names 4 open gaps: adaptive retrieval, cost-aware orchestration, formal trajectory eval, oversight for autonomous loops. PA-AKG addresses the oversight gap by making provenance first-class.
Think-on-Graph 3.0 (ToG-3)
arXiv, Sept 2025		 Sep 2025 Multi-agent dual-evolving context retrieval over heterogeneous graphs. Constructor / Retriever / Reflector / Responser agents. Focuses on retrieval quality, not provenance propagation. No verifiable evidence chains across agents.
Youtu-GraphRAG
arXiv, Aug 2025		 Aug 2025 Vertically unified agents for graph RAG; seed schema, dual community detection, hierarchical knowledge tree. Schema-driven retrieval. No claim-level audit trail.
GraphTracer
arXiv, Oct 2025		 Oct 2025 Information Dependency Graphs for failure attribution in multi-agent deep search. Closest comparator. But diagnostic (after-the-fact) — not runtime-enforcing. PA-AKG enforces provenance during retrieval/generation.
TeaRAG
arXiv, Nov 2025		 Nov 2025 Token-efficient agentic RAG with knowledge association graph + iterative DPO. Efficiency-focused. No provenance propagation contract between agents.
Graph Counselor
arXiv, Jun 2025		 Jun 2025 Multi-agent adaptive graph exploration; Planning / Thought / Execution agents with self-reflection. Reasoning-quality focus. Source attribution is best-effort, not typed.
SPAR (scholar paper retrieval)
arXiv, Jul 2025		 Jul 2025 Multi-agent academic search with RefChain-based query decomposition. Domain-specific (academic search). PA-AKG is domain-agnostic with SE as case study.
AgREE (agentic KG completion)
arXiv, Aug 2025		 Aug 2025 Agentic framework for KG triplet construction on emerging entities; iterative retrieval + multi-step reasoning. Single-agent loop. No verifiable chain for downstream consumers.
AgenticSZZ
arXiv, Feb 2026		 Feb 2026 Temporal Knowledge Graphs + LLM agents for bug-inducing commit identification (SE domain). Direct SE-application precedent. PA-AKG generalizes the pattern with provenance as protocol primitive.
RepoUnderstander
arXiv, Jun 2024		 Jun 2024 Repository knowledge graph + MCTS for whole-repo navigation by SE agents. Single-agent. PA-AKG is multi-agent with typed message provenance.
Agentic RAG — Survey
arXiv, Jan 2025		 Jan 2025 Field survey of agentic RAG design patterns. Reference. PA-AKG positions provenance as a missing first-class primitive in the survey's taxonomy.

AWS hooks for the proposal #aws-hooks

The CFP requires "a list of AWS ML tools to be used." Bedrock's current stack maps almost 1:1 to PA-AKG's architecture, which makes the AWS-fit story unusually clean. Verified against Bedrock docs (May 2026):

Amazon Bedrock Agents — Multi-Agent Collaboration
Supervisor agent breaks down requests into sub-agent tasks. Directly maps to PA-AKG's orchestrator coordinating retrieval / curation / validation specialists.
Amazon Bedrock Knowledge Bases — Hybrid Search
Built-in vector + lexical hybrid retrieval over unstructured sources; built-in citation/source attribution. The retrieval layer of PA-AKG runs here directly.
Bedrock KB — Neptune Analytics graphs
Managed graph backend for the knowledge graph layer. PA-AKG's KG-curation agent reads / writes here. No custom graph infra to stand up.
Bedrock KB — structured data (NL → SQL)
Relational store integration with NL-to-SQL conversion. Supports the "relational" leg of the CFP's hybrid retrieval welcome topic.
Bedrock — Reranking models
Provenance-weighted reranking. Provenance scores feed into rerank as a typed signal — novel use of an existing primitive.
Bedrock — Action Groups
Tool-invocation surface for validation agents (e.g., call a static-analysis tool, call a test runner). Each invocation tagged with provenance.
Amazon SageMaker (training)
Optional: fine-tune a small provenance-routing model on validation traces. Spec calls this out as a Year-1 stretch goal, not a critical path item.

Open-source contribution (mandatory CFP element)

The CFP requires "list of open-source tools to be contributed." Three candidates evaluated; final decision pending PI sign-off:

  • PA-AKG reference implementation — full agent stack (retrieval / KG-curation / validation) on Bedrock. Highest-value deliverable; biggest engineering scope.
  • Provenance evaluation harness — benchmark + eval suite specifically for provenance-aware agentic retrieval. Smaller scope, defensible.
  • Extend a Brown-lab tool (AutoPyDep / AutoPrint) with provenance-aware retrieval. Narrowest scope, leverages prior infra.

Recommendation pending: PA-AKG reference impl + lightweight eval harness, with case-study integration into an existing Brown-lab tool as the empirical anchor.
