Title

Agentic Knowledge Graphs for AI-Accelerated Engineering with Generalizable Simulation-in-the-Loop Validation

📄 Download v4e — Simulation-Grounded Validation (latest) 📄 Download v4d — Synthesis (domain-agnostic + VVUQ + engineering-agent citations) 📄 Download v4c — Construction-Only + VVUQ 📄 Download v4b — Construction + Transportation 📄 Download v4 — 4-Domain Original

Core Idea

Domain-agnostic framework enabling LLM-powered agents to reason over structured engineering knowledge, interact with physics-based simulations, and validate outputs against domain constraints. Three generalizable layers: domain ontology, simulation integration, and constraint validation.

Keywords

Agentic knowledge graphs Simulation-in-the-loop validation VVUQ Retrieval-augmented generation Physics-informed machine learning Construction AI Building Information Modeling (BIM) Finite element analysis Constraint-aware planning Uncertainty quantification Multi-agent orchestration Knowledge graph engineering

Potential Reframing Under Consideration

Under review: Narrowing from 3 generic domains to 2 concrete applications sharing a VVUQ (Verification, Validation, and Uncertainty Quantification) backbone. This addresses feasibility concerns and strengthens advisor alignment.

Application 1: Construction — Material Takeoff with Computer Vision

Uses computer vision for automated material takeoff from construction plans/drawings. Feeds into BOM reasoning and cost estimation. Ties directly into the advisor's construction.ai initiative. KG layer encodes construction domain knowledge (materials, assemblies, building codes).

Application 2: Transportation — Digital Twin

Builds a digital twin for traffic system modeling and optimization. Agent reasons over traffic flow models, network topology, and sensor data. KG layer encodes transportation domain knowledge (network constraints, flow equations).

Shared Component: VVUQ

Both applications share VVUQ as the simulation/analysis backbone:

  • Construction: beam load analysis, structural integrity verification
  • Transportation: traffic flow simulation, capacity analysis
  • Thermal (cross-domain): heat transfer analysis, HVAC sizing

VVUQ provides the rigorous validation layer that distinguishes this from pure LLM approaches and gives the framework a clear scientific identity.

Three Layers (Current Abstract)

  • Domain Ontology Layer — Engineering KGs encoding entities, constraints, physical laws, and design rules as pluggable, domain-specific components
  • Simulation Integration Layer — PDE solvers, FEA tools, and domain simulators exposed as agent-callable tools for simulation-in-the-loop validation
  • Constraint Validation Layer — Automatic verification of agent-generated designs against physical and regulatory constraints

Research Questions

  1. How can engineering domain ontologies be represented as KGs enabling both LLM-based reasoning and structured constraint queries?
  2. How should agents orchestrate across retrieval, graph reasoning, and simulation calls to solve multi-step engineering problems?
  3. What simulation-in-the-loop validation mechanisms ensure agent outputs satisfy physical laws and domain constraints?
  4. Does a domain-agnostic framework achieve competitive accuracy vs. domain-specific systems while reducing development cost?
