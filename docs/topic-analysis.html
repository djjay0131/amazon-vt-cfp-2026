CFP Scope & Research Alignment

The Amazon-VT CFP 2026-2027 invites proposals across 9 major AI topic areas. Our research capabilities overlap with multiple high-priority tracks—especially those referencing knowledge grounding, multi-agent systems, agent orchestration, memory-augmented systems, and AI-accelerated engineering.

This is not a marginal fit—this CFP is unusually aligned with our research direction. The strongest overlaps are in:

  • Knowledge grounding & multi-source fusion
  • Agentic AI & multi-agent orchestration
  • Agent evaluation & validation
  • AI-accelerated science & engineering
  • Efficient retrieval over heterogeneous knowledge sources

Ranked Proposal Directions

Three candidate directions ranked by funding probability, differentiation, Amazon relevance, and 1-year feasibility. Click each card to expand.

🥇 Provenance-Aware Agentic Knowledge Graph Systems for Reliable Multi-Source Grounded AI Selected ▶
Primary: Knowledge Grounding Secondary: Agentic AI
Why #1? Direct CFP language match. Differentiates from “just another RAG proposal”. Amazon relevance: Enterprise AI + Agents. Existing infrastructure and case studies.

Motivation

Current RAG systems rely primarily on vector similarity matching, which provides relevance but lacks structured reasoning, provenance tracking, and verifiable evidence chains. As agentic systems become autonomous, the lack of grounded traceability increases risks of hallucination, misinformation, and unverifiable decision-making.

Research Questions

  1. How can heterogeneous knowledge sources (KGs, databases, streaming data, documents) be fused into a unified structured reasoning substrate?
  2. How can agents dynamically select structured vs vector retrieval pathways?
  3. How can provenance chains be automatically generated, verified, and audited?
  4. How does structured grounding affect reasoning accuracy, robustness, and inference efficiency?

Technical Approach

  • Hybrid KG + vector retrieval architecture
  • Agent orchestration workflows with validation nodes
  • Provenance generation mechanisms
  • Evaluation benchmarks: vector-only RAG vs hybrid RAG vs fully structured KG agents

Expected Contributions

  • New architecture for agentic knowledge grounding
  • Provenance-aware reasoning benchmark
  • Evaluation metrics for grounded multi-agent systems
  • Open-source reference implementation
🥈 A Structured Validation Framework for Knowledge-Grounded and Multi-Agent AI Systems Alternative ▶
Primary: Agentic Evaluation Secondary: Responsible AI
Why #2? High strategic value. Amazon cares deeply about evaluation. Leverages validation matrix research. Safer and more focused than broad agent proposal.

Motivation

While generative AI evaluation has focused on accuracy benchmarks and jailbreak robustness, there is limited infrastructure for evaluating structured reasoning correctness, multi-agent coordination reliability, knowledge grounding integrity, and provenance traceability.

Research Questions

  1. What dimensions define reliable grounded agent behavior?
  2. How do structured grounding mechanisms impact agent robustness?
  3. How can validation loops be embedded into agent execution pipelines?
  4. How do we benchmark multi-agent failure modes?

Technical Approach

  • Validation matrix taxonomy
  • Benchmark tasks across reasoning domains
  • Agent failure scenario simulations
  • Structured logging and provenance scoring
  • Robustness evaluation under adversarial perturbations

Expected Contributions

  • First structured evaluation framework for KG-grounded agents
  • Metrics for agent traceability and reliability
  • Benchmarks for multi-agent coordination integrity
🥉 Agentic Knowledge Graphs for AI-Accelerated Engineering and Digital Twin Systems Alternative ▶
Primary: AI for Science/Engineering Secondary: Agentic AI
Why #3? Extremely compelling narrative. Aligns with Construction-AI + VTSI. Most visionary, highest impact. Slightly riskier—requires strong engineering framing.

Motivation

Engineering workflows involve structured constraints, physical laws, and simulation models. Current generative systems struggle with physics-grounded reasoning and constraint validation.

Research Questions

  1. How can generative AI interact with physical simulation systems?
  2. How can agents reason over engineering constraints?
  3. How can digital twin feedback update agent knowledge graphs?
  4. What are efficiency gains from structured reasoning vs unstructured LLM prompts?

Technical Approach

  • Structured engineering knowledge graphs
  • Simulation model integration (heat transfer, construction planning)
  • Multi-agent reasoning pipelines
  • Engineering decision accuracy evaluation

Expected Contributions

  • Framework for AI-accelerated engineering reasoning
  • Digital twin + agent integration architecture
  • Quantitative comparison vs prompt-only methods

Open Questions for Due Diligence

Questions to Revisit Before Submission

  1. Competitive landscape: What are other VT labs likely to propose? Overlap risk?
  2. Amazon strategic priorities: Which topics does Amazon care most about this year?
  3. Novelty depth: For PA-AKG, how does it differ from existing KG-RAG systems (e.g., GraphRAG, KGQA)? Need concrete differentiation beyond “provenance.”
  4. Feasibility: Can this produce meaningful results in 1 year with doctoral fellowship resources?
  5. Combinability: Can elements of Proposal 2 (evaluation) be folded into Proposal 1 as a deliverable?
  6. PI alignment: Which direction best matches the PI’s existing publication record and funded work?
  7. Amazon-specific use cases: Which proposal best maps to Amazon product/service areas (Alexa, AWS, retail, logistics)?
