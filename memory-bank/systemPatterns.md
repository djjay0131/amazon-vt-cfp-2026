# System Patterns

## Proposal Development Workflow

### Design-First Pattern
All proposal content follows the construction/ workflow:
1. Design document created and reviewed
2. Sprint tasks defined from design
3. Content written per sprint plan
4. Validated against CFP requirements

### Memory-Bank Pattern
Living documentation maintained throughout:
- Decisions recorded as ADRs in `architecturalDecisions.md`
- Progress tracked in `progress.md`
- Context updated in `activeContext.md` each session
