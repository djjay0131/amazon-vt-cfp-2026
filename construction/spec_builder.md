# Spec Builder — Design Document Template

Use this template when creating new design documents in `construction/design/`.

---

## Template

```markdown
# Design: [Title]

## Objective
_What is this design for? What problem does it solve?_

## CFP Alignment
- **Primary Topic**: _Which CFP topic area does this address?_
- **Secondary Topic**: _Optional secondary topic area_
- **Relevance**: _Why is this relevant to Amazon-VT Initiative goals?_

## Background
_Context, prior work, and motivation. What literature supports this?_

## Proposed Approach
_Detailed description of the research approach or proposal section content._

## Key Contributions
_What is novel? What advances the state of the art?_

## Feasibility
- **Timeline**: _Can this be completed in 1 year?_
- **Resources**: _What is needed (compute, data, personnel)?_
- **Risks**: _What could go wrong? Mitigations?_

## References
_Key papers and resources._

## Acceptance Criteria
- [ ] _Criterion 1_
- [ ] _Criterion 2_

## Status
- **Created**: YYYY-MM-DD
- **Last Updated**: YYYY-MM-DD
- **State**: Draft | Review | Approved | Superseded
```

---

## Usage

1. Copy template to `construction/design/<name>.md`
2. Fill in all sections
3. Review against CFP requirements
4. Get approval before drafting proposal content
